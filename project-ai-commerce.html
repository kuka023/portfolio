Tech Decisions

核心技术选型对比

为什么这样选——每个模块的对比依据与业务 KPI 对齐逻辑。

🧠
文案生成 LLM

开源微调 vs 闭源 API

✗ 闭源 API
高质量但成本高、数据出域、风格漂移难固化
✓ 开源微调
可私有化、风格可固化、边际成本低、可 A/B
选择理由：85% 平台风格匹配率 + 高并发 + 成本可控，用 Qwen 基座 + 指令微调 + 模板词库；闭源保留为兜底
🖼️
商品图生成

SDXL+ControlNet vs 闭源文生图

✗ MJ / DALL·E
创意强但不可复现、版权模糊、批量难工程化
✓ SDXL + LoRA
强可控、可复现、可批量、风格一致、可私有化
选择理由：固定 seed/构图模板保证批量一致性；LoRA 沉淀品牌资产，满足 2 分钟批量出图要求
✂️
商品抠图

U²-Net vs SAM（通用大模型）

✗ SAM
泛化强但推理重，难满足"秒级/高并发"要求
✓ U²-Net
速度/精度/资源平衡最优，边缘干净，工程稳定
选择理由：规则拍摄商品图分布下 mIoU 足够，秒级完成；极端场景触阈 → 升级策略或人工审核
🎬
短视频生成

模板流水线 vs 端到端文生视频

✗ 端到端生视频
创意强但渲染慢、稳定性差，难达"2分钟出片"
✓ 模板化剪辑
确定性强、渲染快、可规模化、品牌模板可复用
选择理由：完全贴合 KPI（时延/成本/可控/复用）；Runway Gen-3 等前沿模型作为灰度升级通道

Product Process

产品推进阶段

01

需求调研与定义

深入理解商家实际工作流，避免技术驱动脱节。

  • 10-15 家商家深度访谈，挖掘真实痛点
  • 竞品分析（Canva / PhotoRoom / CapCut）
  • 按品类 × 平台分类场景需求
  • 确定商家对生成质量的接受底线
02

系统流程设计

将需求拆解为可执行的产品流程与功能架构。

  • 五步端到端流水线设计
  • 信息架构（Dashboard / 工作台 / 素材库）
  • 质量评估体系与回退机制
  • 多平台风格模板规格定义
03

技术方案选型

以 KPI 为锚点，逐类对比并给出"为什么这样选"。

  • 文案 LLM：开源微调 vs 闭源 API
  • 图像：SDXL+ControlNet vs MJ/DALL·E
  • 抠图：U²-Net vs SAM
  • 视频：模板化流水线 vs 端到端文生视频
04

评测与迭代闭环

建立离线 + 在线双层评估，持续自动化迭代。

  • 离线：CLIP Score / 要点覆盖率 / 敏感词率
  • 在线：CTR / 加购率 / 二改率
  • 双层过滤（禁词 + 图像瑕疵）+ 质量阈值回退
  • A/B 测试 AI 生成 vs 人工制作效果

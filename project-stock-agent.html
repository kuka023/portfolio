Problem & Solution

发现问题，定义场景

❌ 用户痛点
  • •需同时打开多个平台手动取数，流程割裂，决策滞后
  • •传统软件功能复杂，学习成本高，普通投资者难上手

✅ 产品方案
  • •自然语言对话入口，一句话触发 8 类数据工具并行分析
  • •多 LLM 接入，按场景自由选择模型，平衡质量与成本
  • •流式输出 + 会话记忆，支持多轮深度追问
  • •飞书机器人异动推送，主动触达用户，从被动查询转为主动提醒

Feature Deep Dive

股票筛选 · 两层过滤引擎

A 股有 5000+ 只股票，用户的第一个问题不是"这只股票怎么样"，而是"今天该看哪只"。筛选功能解决的是发现机会的入口问题。

第一层 · 实时行情（毫秒级，全市场）
估值PE 范围 · PB 上限
行情涨跌幅区间 · 量比
规模市值范围 · 换手率
板块仅主板（排除创业/科创/北交所）
第二层 · 历史行为（对候选股逐只拉取，10 只并发）
避免追高昨日未涨停
主力活跃近一个月有过涨停
趋势向上近三个月创新高
候选超 80 只时历史条件自动停用 + 提示用户收窄 → 主动保护机制
自然语言示例 → 自动转化为筛选规则
用户说： "找 PE 低于 20、ROE 超过 15% 的主板股票"
用户说： "今天涨幅 2%–5%、换手率大于 3% 的股票"
用户说： "近三个月创新高、近一个月有过涨停的票"
Feature Deep Dive

异动监控 · 两段式扫描架构

用户发现了目标股票，但不可能一直盯着盘。异动监控负责在关键时刻主动找到用户，而不是让用户自己去捕捉时机。

架构核心：把数据获取成本前置，把告警响应成本降到最低
盘前 9:25
建立背景档案
对所有监控股票并行拉取 4 个维度：基本面 + 技术面 + 资金流向 + 近期新闻。结果缓存内存，作为全天分析的"底稿"，避免重复拉取。
交易时段
每 30 秒轻量轮询
只检查价格和量比两个指标，不调用重型接口。触发阈值时才启动分析，分析时直接引用底稿，不重复拉取数据。
午后 12:55
部分底稿刷新
只更新资金流向和新闻（变化快），基本面和技术面不重拉（变化慢、成本高）。差异化更新策略降低 API 消耗。
17:00
自动关闭轮询
收盘后停止所有无效轮询，不产生多余的 API 调用与系统开销。

主力行为识别 · 超大单 Delta 检测

触发异动后不只是推送"涨了多少"，而是判断主力在做什么，并每 3 分钟检测超大单变化量——即使价格未大幅波动，悄悄建仓或出货也能被发现：

🚀
拉升期
超大单+大单同向净流入，价格大涨，量比 >1.5
🟢
建仓期
超大单净流入，价格横盘或小涨
🔴
出货期
超大单净流出，价格高位上涨，散户在买
⚠️
对倒出货
超大单卖+大单买，左手倒右手，骗散户进场
🟡
洗盘期
超大单小幅流出，价格小跌，震仓清浮筹
